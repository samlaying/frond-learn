# 前端学习路线

这个仓库用于记录一条以“学会并做出来”为目标的前端学习路线。内容分为三根支柱：

1. **教程内容**：按阶段学习基础知识，并用项目验证理解。
2. **学习方式**：用可复习、可输出、可迭代的方法推进学习。
3. **代码实践**：把现有 demo 当作拓展练习，重点放在拆解、改写和迁移，而不是只看效果。

可直接运行的实战入口在 [`practice/frontend-architecture/`](/Users/sam/03-Code/frond-learn/practice/frontend-architecture/)。

## 一、教程内容：五个阶段

### 阶段 1：网页基础

目标是能独立写出结构清晰、语义合理的静态页面。

- HTML 语义化、表单、可访问性基础
- CSS 选择器、盒模型、字体、颜色、背景和渐变
- Flex、Grid、定位与响应式布局
- 项目：个人介绍页或产品落地页

完成标准：页面能在不同宽度下正常显示，并能解释主要布局为什么这样写。

### 阶段 2：JavaScript 与浏览器

目标是理解页面交互，而不是只会套用组件。

- 变量、函数、数组、对象、模块和错误处理
- DOM、事件、表单校验、异步请求
- 浏览器存储、网络请求和基础调试
- 项目：待办清单、搜索页或小型数据面板

完成标准：能从需求拆出状态、事件和渲染逻辑，并定位常见运行时错误。

### 阶段 3：工程化与组件开发

目标是把页面组织成可维护、可复用的代码。

- npm、版本控制、目录组织和环境变量
- React（或同类框架）的组件、状态、路由和数据流
- TypeScript 基础与类型建模
- 补充教程：[`一张图讲清前端架构`](./docs/frontend-architecture.md)（SPA、分层、Event Bus、IndexedDB、Web Worker 与 PWA）
- 项目：一个包含多个页面和接口请求的前端应用

完成标准：能说明组件边界、数据流向和关键类型，项目可以被别人启动和修改。

### 阶段 4：视觉与交互实践

目标是把基础能力转化为有质量的界面实现。

- 动画、过渡、响应式细节和移动端适配
- 阴影、玻璃、拟态、3D、视差等视觉效果
- 性能、可访问性和降级处理
- 项目：完成一个有明确主题的交互式页面

拓展入口：[`archive/shicao/README.md`](./archive/shicao/README.md)。其中的 demo 是实践素材，不替代前面的基础学习。

### 阶段 5：综合项目与作品沉淀

目标是完成一个可以展示、复盘和持续迭代的作品。

- 从需求、信息架构到页面实现
- 接入真实数据，处理加载、空状态和错误状态
- 测试、性能优化、部署和项目说明
- 项目：独立完成一个完整的前端作品

完成标准：项目有清晰 README、可运行地址或录屏、技术取舍说明，以及一次完整复盘。

## 二、学习方式：每个主题都走一遍闭环

1. **先理解**：用教程建立概念模型，记录“它解决什么问题”。
2. **再模仿**：不看答案重写最小例子，确认自己能复现。
3. **再改造**：改变内容、布局或交互，观察哪些代码必须随之变化。
4. **再应用**：把知识放进当前阶段的项目，而不是停留在孤立练习。
5. **再输出**：用短笔记或代码注释说明原理、取舍和踩坑。
6. **定期复盘**：每周保留一个可运行成果，并列出下周只做一到三个改进点。

遇到归档区中的 demo 时，建议按“看效果 → 猜结构 → 关闭参考重写 → 修改参数 → 迁移到自己的项目”使用。`prompt.md` 是实现提示和拆解材料，应作为拓展参考，不应当被当成教程主线直接照抄。

## 三、代码实践：现有 demo 的定位

`archive/` 保存已经完成或正在整理的实战实现，不属于教程正文。`archive/shicao/` 是视觉效果和交互 demo 集合，适合在阶段 4 及之后使用。它可以帮助练习：

- 阅读并拆解 HTML、CSS、JavaScript
- 比较不同实现方案的优缺点
- 将单个效果提取到自己的页面
- 为效果补充响应式、键盘操作和降级方案

实践记录建议包含：实现目标、使用技术、自己改了什么、遇到的问题，以及下一步可以如何优化。

## 归档区

归档区目前包含以下实战项目：

- `archive/shicao/`：视觉与交互效果实践库
- `archive/1-game/`：小游戏实现
- `archive/ppt-html/`：HTML 展示页面实现
- `archive/planets-animated-website/`：动画网站实现
- `archive/loginsixteen-master/`：页面复刻与样式实现

这些项目不构成学习顺序。学习时根据教程当前阶段选择对应项目拆解、重写和迁移即可。

## 使用原则

- 不把“看过教程”当成“学会”，以能解释和改造为准。
- 不追求一次学完所有技术，当前阶段只保留必要内容。
- 每个阶段都要有项目成果，项目优先于收藏更多资料。
- 遇到不懂的效果，先缩小问题并做最小实验，再回到完整项目。

用消息把模块连起来

这章练的是“不是直接调用，而是先发消息、别人自己来接”的思路。 你会看到一个最小 Event Bus，支持 `on`、`emit`、`off`，然后让一个模块发消息，另一个模块收消息。

你可以把它理解成：A 模块不认识 B 模块，只认识一个名字叫事件类型的“消息频道”。 谁订阅了这个频道，谁就会收到通知。

判断标准：
`emit` 只是广播消息，不负责处理业务； `on` 是订阅； `off` 是取消订阅。
页面初始化完成，等待发送消息。
你要记住的关键词
  • 一个模块发消息，不需要知道谁会接。
  • 多个监听者可以同时响应同一条消息。
  • 取消订阅后，这个监听者就不再收到通知。
  • 消息类型越明确，代码越好读。
